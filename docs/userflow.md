## 0. 루트 페이지 접근

### 0.1 입력
- 사용자가 브라우저 주소창에 도메인 루트 URL 입력 (`/`)
- 또는 북마크/즐겨찾기에서 루트 페이지 접근

### 0.2 처리
- **인증 상태 확인**
  - Django 세션 검증
  - `request.user.is_authenticated` 확인

- **분기 처리**
  - 미인증 사용자: 로그인 플로우로 진행 (Section 2 참조)
  - 인증된 사용자: 대시보드로 리디렉션

### 0.3 출력
- **미인증 사용자 (is_authenticated = False)**
  - 로그인 폼 렌더링 (`/login`과 동일한 템플릿 사용)
  - URL은 `/`로 유지
  - 템플릿: `authentication/login.html`

- **인증된 사용자 (is_authenticated = True)**
  - HTTP 302 리디렉션: `/dashboard`
  - 로그인 폼 표시하지 않음
  - Django `redirect()` 함수 사용

### 0.4 엣지케이스
- 만료된 세션: 미인증 사용자로 간주, 로그인 폼 표시
- 세션 쿠키 없음: 미인증 사용자로 간주
- 비활성화된 계정 (status='inactive'): 미인증으로 간주, 로그인 시도 시 에러 표시
- 승인 대기 계정 (status='pending'): 미인증으로 간주, 로그인 시도 시 승인 대기 메시지

## 1. 회원가입

### 1.1 입력
- 회원가입 페이지 접근
- 사용자 정보 입력 (이름, 이메일, 비밀번호, 비밀번호 확인, 소속 부서, 직책)
- 회원가입 버튼 클릭

### 1.2 처리
- **입력값 검증**
  - 필수 항목 누락 여부 확인
  - 이메일 형식 유효성 검사
  - 비밀번호 정책 검증 (최소 4자)
  - 비밀번호 일치 여부 확인

- **중복 검사**
  - 이메일 중복 여부 데이터베이스 조회
  - 중복 시 프로세스 중단

- **데이터 저장**
  - 비밀번호 해싱 처리
  - 사용자 정보를 데이터베이스에 저장
  - 사용자 상태를 "승인 대기(pending)"로 설정
  - 생성 일시 기록

### 1.3 출력
- **성공 시**
  - 팝업 메시지 표시: 관리자 승인 대기 안내
  - 팝업 확인 후 로그인 페이지로 리디렉션

- **실패 시 (입력값 오류)**
  - 해당 필드에 오류 메시지 표시
  - 입력한 데이터 유지 (비밀번호 제외)
  - 페이지 유지

- **실패 시 (이메일 중복)**
  - 이메일 중복 안내 메시지 표시
  - 로그인 페이지로 이동 링크 제공
  - 비밀번호 찾기 링크 제공

- **실패 시 (서버 오류)**
  - 시스템 오류 안내 메시지 표시
  - 재시도 안내

### 1.4 엣지케이스
- 네트워크 타임아웃: 중복 가입 방지를 위한 이메일 기반 체크
- 특수문자가 포함된 이름: 허용 가능한 특수문자 범위 정의 및 검증
- 동시 가입 요청: 데이터베이스 유니크 제약조건으로 중복 방지
- 매우 긴 입력값: 각 필드별 최대 길이 제한 및 검증

## 2. 로그인

### 2.1 입력
- 로그인 페이지 접근
- 이메일과 비밀번호 입력
- 로그인 버튼 클릭

### 2.2 처리
- **입력값 검증**
  - 필수 항목 누락 여부 확인
  - 이메일 형식 유효성 검사

- **사용자 인증**
  - 이메일로 사용자 데이터베이스 조회
  - 사용자 존재 여부 확인
  - 비밀번호 해시 비교 검증

- **사용자 상태 확인**
  - 사용자 상태(status) 확인
  - pending 상태: 승인 대기 중
  - inactive 상태: 비활성화된 계정
  - active 상태: 정상 계정

- **세션 생성 (active 상태인 경우)**
  - Django 세션 생성 (사용자 id, role, email 등 세션에 저장)
  - 세션 쿠키를 클라이언트에 전달
  - 로그인 시간 기록
  - (사유: MVP 범위에서는 Django 기본 세션 인증 사용)
  - (2단계 로드맵: API 전환 시 JWT 기반 인증 도입 예정)

### 2.3 출력
- **성공 시 (active 상태)**
  - 대시보드 페이지(/dashboard)로 리디렉션
  - Django 세션 쿠키 자동 저장 (브라우저)

- **실패 시 (pending 상태)**
  - 팝업 메시지 표시: 관리자 승인 대기 중 안내
  - 로그인 페이지 유지

- **실패 시 (inactive 상태)**
  - 계정 비활성화 안내 메시지 표시
  - 관리자 문의 안내
  - 로그인 페이지 유지

- **실패 시 (이메일 또는 비밀번호 오류)**
  - 로그인 실패 메시지 표시 (보안을 위해 구체적인 오류 항목 명시하지 않음)
  - 입력한 이메일 유지
  - 비밀번호 필드 초기화

- **실패 시 (입력값 오류)**
  - 해당 필드에 오류 메시지 표시
  - 입력한 데이터 유지

- **실패 시 (서버 오류)**
  - 시스템 오류 안내 메시지 표시
  - 재시도 안내

### 2.4 엣지케이스
- 연속 로그인 실패: 일정 횟수(예: 5회) 이상 실패 시 계정 일시 잠금 및 대기 시간 부여
- 이미 로그인된 상태에서 접근: 대시보드로 자동 리디렉션
- 만료된 토큰으로 접근: 로그인 페이지로 리디렉션 및 재로그인 안내
- 네트워크 타임아웃: 재시도 안내 메시지 표시
- SQL Injection 시도: 파라미터화된 쿼리로 방어

## 3. 엑셀 파일 업로드 (Django Admin 활용)

### 3.1 입력
- 로그인된 관리자 권한 사용자가 Django 관리자 페이지(/admin) 접근
- 해당 데이터 모델 선택 (학과별 KPI, 논문 게재 목록, 연구비 집행 데이터, 학생 명단)
- "추가(Add)" 버튼 클릭
- 파일 업로드 필드에서 엑셀/CSV 파일 선택
- "저장(Save)" 버튼 클릭
- (사유: MVP 개발 속도 극대화를 위해 별도 UI 개발을 지양하고, Django의 강력한 기본 기능 활용)

### 3.2 처리
- **권한 확인**
  - Django 세션 인증 검증
  - 사용자 role 확인 (staff 또는 superuser만 Django Admin 접근 가능)
  - 권한 없을 시 로그인 페이지로 리디렉션

- **파일 검증**
  - 파일 존재 여부 확인
  - 파일 확장자 검증 (.xlsx, .xls, .csv)
  - 파일 크기 확인 (최대 50MB)
  - Django Admin 폼 검증 수행

- **파일 처리**
  - Django 모델의 save() 메서드 또는 시그널에서 파일 처리
  - Pandas를 사용한 파일 파싱 (엑셀/CSV)
  - 모델 타입에 맞는 스키마로 데이터 검증

- **데이터 타입별 검증**
  - **학과별 KPI**: 평가년도, 단과대학, 학과, 졸업생 취업률, 전임교원 수, 초빙교원 수, 연간 기술이전 수입액, 국제학술대회 개최 횟수
  - **논문 게재 목록**: 논문ID, 게재일, 단과대학, 학과, 논문제목, 주저자, 참여저자, 학술지명, 저널등급, Impact Factor, 과제연계여부
  - **연구비 집행 데이터**: 집행ID, 과제번호, 과제명, 연구책임자, 소속학과, 지원기관, 총연구비, 집행일자, 집행항목, 집행금액, 상태, 비고
  - **학생 명단**: 학번, 이름, 단과대학, 학과, 학년, 과정구분, 학적상태, 성별, 입학년도, 지도교수, 이메일
  - 필수 컬럼 존재 여부 확인
  - 데이터 타입 검증 (숫자, 문자열, 날짜 등)
  - 중복 데이터 확인 (학과별 KPI는 평가년도+학과, 논문은 논문ID, 연구비는 집행ID, 학생은 학번 기준)

- **데이터 저장**
  - 검증 통과 시 데이터베이스에 일괄 저장 (bulk_create 활용)
  - 중복 데이터 처리 옵션 적용 (업데이트 또는 무시)
  - UploadHistory 모델에 업로드 이력 자동 기록 (user, file_name, file_size, upload_date, status, 데이터 타입, 저장된 행 수)
  - 성공 상태로 기록

- **오류 처리**
  - 검증 실패 시 Django의 ValidationError 발생
  - 오류 행/열 번호 및 내용을 에러 메시지에 포함
  - 실패 상태로 업로드 이력 기록 (error_message 포함)

### 3.3 출력
- **성공 시**
  - Django Admin의 성공 메시지 표시: "파일 업로드 및 데이터 처리가 완료되었습니다. N건의 데이터가 저장되었습니다."
  - Django Admin 목록 페이지로 리디렉션
  - 업로드된 데이터를 Django Admin 목록에서 즉시 확인 가능
  - UploadHistory 모델에 자동으로 이력 기록됨

- **실패 시 (권한 없음)**
  - Django Admin 로그인 페이지로 리디렉션
  - "이 페이지에 접근할 권한이 없습니다" 메시지 표시

- **실패 시 (파일 형식 오류)**
  - Django Admin 폼 오류 메시지 표시
  - "지원하지 않는 파일 형식입니다. .xlsx, .xls, .csv 파일만 업로드 가능합니다."

- **실패 시 (파일 크기 초과)**
  - Django Admin 폼 오류 메시지 표시
  - "파일 크기가 제한(50MB)을 초과했습니다."

- **실패 시 (데이터 검증 오류)**
  - Django Admin 폼 상단에 오류 메시지 표시
  - 오류 상세 정보: "데이터 검증 실패 - 3행 '졸업생 취업률' 컬럼: 숫자 형식이어야 합니다"
  - 업로드 폼 유지 (수정 후 재시도 가능)
  - 오류 로그가 UploadHistory 모델에 자동 기록됨

- **실패 시 (서버 오류)**
  - Django Admin 오류 페이지 표시
  - "시스템 오류가 발생했습니다. 관리자에게 문의하세요."
  - 에러 로그에 자동 기록

### 3.4 엣지케이스
- 빈 파일 업로드: Django Admin 폼 검증 오류 - "파일에 데이터가 없습니다"
- 헤더가 없는 파일: 첫 번째 행을 헤더로 인식, 컬럼명 불일치 시 검증 오류 표시
- 컬럼 순서가 다른 파일: 컬럼명 기반으로 자동 매핑 (Pandas 활용)
- 선택 컬럼 누락 (예: Impact Factor, 지도교수): NULL 허용 (모델 정의에 따름)
- 매우 큰 파일 (50MB 이하지만 수만 행): Django의 요청 타임아웃 내에서 처리 (필요시 Celery 등 비동기 처리 고려)
- 동일 데이터 중복 업로드: 유니크 제약조건에 따라 자동 처리 (업데이트 또는 오류)
- 업로드 중 네트워크 끊김: Django의 기본 타임아웃 처리, 재시도 필요
- 특수문자가 포함된 파일명: Django가 자동으로 안전한 파일명으로 변환
- 날짜 형식 불일치: Pandas의 to_datetime() 함수로 다양한 형식 자동 파싱
- CSV 인코딩 문제: Pandas의 자동 인코딩 감지 또는 encoding 파라미터 명시

**중요:** (2단계 로드맵: 사용자 친화적인 드래그 앤 드롭 업로드 페이지 개발 예정. 현재는 MVP 개발 속도 극대화를 위해 Django Admin 활용)

## 4. 대시보드 조회

### 4.1 입력
- 로그인된 사용자가 대시보드 페이지(/dashboard) 접근
- 선택적으로 기간 필터 설정 (연도, 학기)
- 선택적으로 새로고침 버튼 클릭

### 4.2 처리
- **권한 확인**
  - Django 세션 인증 검증
  - 사용자 role 및 소속 확인
  - 로그인 상태 확인

- **데이터 조회**
  - 기본값: 최근 데이터 조회 (최근 학기 또는 최근 연도)
  - 필터 설정 시 해당 기간의 데이터 조회
  - 사용자 role에 따른 데이터 접근 범위 결정
    - 관리자: 전체 데이터
    - 경영진: 전체 데이터
    - 일반 사용자: 소속 부서 데이터만

- **KPI 계산**
  - 학과별 KPI 데이터 집계 (취업률 평균, 총 교원 수, 기술이전 수입액 합계, 학술대회 개최 횟수 합계)
  - 논문 게재 수 집계 (총합, 학과별, 등급별)

  - 재시도 버튼 제공

### 4.4 엣지케이스

- **필터 파라미터 처리**
  - 기본값 설정: 최근 3개년, 전체 학과, 전체 지원기관, 전체 집행항목, 전체 상태
  - 사용자가 선택한 필터값 적용
  - 사용자 role에 따른 데이터 접근 범위 제한
    - 일반 사용자: 소속 학과만 조회 가능
    - 관리자/경영진: 전체 학과 조회 가능

- **데이터 조회**
  - 필터 조건에 맞는 연구비 집행 데이터 데이터베이스 조회
  - 데이터 집계 및 그룹핑
    - 학과별 집계
    - 연도별 집계
    - 지원기관별 집계
    - 집행항목별 집계
    - 연구책임자별 집계
    - 과제별 집계

- **통계 계산**
  - 총 연구비 (과제별 총연구비 합계)
  - 총 집행금액
  - 평균 집행률 (집행금액/총연구비 * 100, 과제별 계산 후 평균)
  - 학과별 연구비 수주 현황
  - 지원기관별 연구비 분포
  - 집행항목별 지출 현황
  - 연구책임자별 과제 수 및 총연구비
  - 집행상태별 분포
  - 연도별 연구비 수주 및 집행 추이

- **차트 데이터 포맷팅**
  - 선택된 차트 타입에 맞는 데이터 구조로 변환
  - 지원기관별, 집행항목별 색상 코드 적용
  - 레이블, 값, 단위 정보 설정 (억원, 만원 등)

### 7.3 출력

  - 필터 옵션 패널 표시 (현재 선택값 표시)
  - 주요 통계 카드 표시 (총 연구비, 총 집행금액, 평균 집행률, 과제 수)
  - 시각화 차트 표시
    - 학과별 연구비 수주 현황 (막대 그래프)
    - 지원기관별 연구비 분포 (파이 차트 또는 도넛 차트)
    - 집행항목별 지출 현황 (막대 그래프)
    - 연도별 연구비 수주 및 집행 추이 (듀얼 축 라인 차트)
    - 연구책임자별 연구비 현황 (막대 그래프, 상위 10명)
    - 과제별 집행률 (막대 그래프 또는 히트맵)

  - 데이터 테이블 뷰 (토글 활성화 시)

    - 정렬 및 검색 기능
    - 페이지네이션
  - 데이터 내보내기 버튼 (CSV, Excel)
  - 차트 이미지 저장 버튼

- **실패 시 (권한 없음)**
  - 로그인 페이지로 리디렉션
  - 세션 만료 안내

  - 데이터 업로드 페이지로 이동 링크 제공 (권한 있는 경우)

- **실패 시 (서버 오류)**
  - 시스템 오류 안내 메시지 표시
  - 재시도 버튼 제공

### 7.4 엣지케이스
- 연구책임자명 검색 시 결과 없음: 검색어 재입력 안내 및 유사 이름 제안
- 필터 조합으로 데이터 없음: 필터 조건 완화 제안 또는 초기화 옵션 제공
- 동일 과제의 여러 집행 건: 과제별로 그룹핑하여 표시 옵션 제공
- 집행률 계산 불가 (총연구비 0): "N/A" 표시
- 집행금액이 총연구비 초과: 초과 집행 경고 표시
- 처리중 상태의 집행 건: 집행률 계산 시 제외 옵션
- 비고가 긴 경우: 툴팁에 전체 내용 표시, 테이블에는 축약
- 매우 큰 금액: 억원 단위로 자동 변환 표시
- 집행항목이 표준화되지 않은 경우: 유사 항목 자동 그룹핑
- 지원기관명 표기 불일치: 표준화된 명칭으로 정규화
- 만료된 토큰: 자동 로그아웃 후 로그인 페이지로 리디렉션
- 차트 렌더링 실패: 대체 데이터 테이블 뷰 제공

## 8. 학생 데이터 시각화 조회

  - 학적상태 선택 (전체, 재학, 휴학, 졸업)
  - 성별 선택 (전체, 남, 여)

  - 사용자가 선택한 필터값 적용
  - 사용자 role에 따른 데이터 접근 범위 제한
    - 일반 사용자: 소속 학과만 조회 가능
    - 관리자/경영진: 전체 학과 조회 가능

- **데이터 조회**

  - 데이터 집계 및 그룹핑
    - 학과별 집계
    - 단과대학별 집계
    - 학년별 집계
    - 과정구분별 집계
    - 학적상태별 집계
    - 성별 집계
    - 입학년도별 집계
    - 지도교수별 집계

- **통계 계산**

### 8.3 출력

  - 필터 옵션 패널 표시 (현재 선택값 표시)
  - 주요 통계 카드 표시 (총 학생 수, 재학률, 휴학률, 남녀 비율)
  - 시각화 차트 표시
    - 학과별 재학생 수 (막대 그래프)

    - 과정구분별 분포 (파이 차트)
    - 학적상태별 분포 (파이 차트 또는 막대 그래프)
    - 입학년도별 학생 수 추이 (라인 차트)
    - 단과대학별 학생 수 비교 (막대 그래프)

    - 정렬 및 검색 기능
    - 페이지네이션
  - 데이터 내보내기 버튼 (CSV, Excel)
  - 차트 이미지 저장 버튼

- **실패 시 (권한 없음)**

  - 소속 학과 데이터만 조회 가능 안내

- **실패 시 (데이터 없음)**

- **실패 시 (서버 오류)**
  - 시스템 오류 안내 메시지 표시
  - 재시도 버튼 제공

### 8.4 엣지케이스
- 지도교수명 검색 시 결과 없음: 검색어 재입력 안내

## 9. 관리자 - 사용자 승인/거부

### 9.1 입력
- 관리자 권한 사용자가 관리자 페이지(/admin) 접근
- 사용자 목록 탭 선택
- 필터 옵션 선택 (전체, 승인 대기, 활성, 비활성)
- 특정 사용자 선택
- 승인 또는 거부 버튼 클릭
- 거부 사유 입력 (거부 시)
- 확인 버튼 클릭

### 9.2 처리
- **권한 확인**
  - Django 세션 인증 검증
  - 사용자 role이 'admin'인지 확인
  - 관리자 권한 없을 시 프로세스 중단

- **필터 파라미터 처리**
  - 기본값: 승인 대기(pending) 상태 사용자 목록
  - 선택한 필터에 따라 사용자 목록 조회

- **사용자 목록 조회**
  - 데이터베이스에서 사용자 정보 조회
  - 필터 조건에 맞는 사용자 목록 반환
  - 사용자 정보: id, 이름, 이메일, 소속 부서, 직책, 가입일, 상태

- **승인/거부 처리**
  - 선택한 사용자 정보 확인
  - **승인 시**
    - 사용자 상태를 'active'로 변경
    - 변경 일시 기록
    - 승인 처리자(관리자) 기록
  - **거부 시**
    - 거부 사유 유효성 검증 (필수 입력)
    - 사용자 상태를 'inactive'로 변경
    - 거부 사유 저장
    - 변경 일시 기록
    - 거부 처리자(관리자) 기록

- **데이터베이스 업데이트**
  - 사용자 정보 업데이트
  - 관리 로그 기록 (누가, 언제, 무엇을, 왜)

### 9.3 출력
- **성공 시 (승인)**
  - 승인 완료 메시지 표시
  - 사용자 목록 자동 갱신
  - 승인된 사용자가 목록에서 제거되거나 '활성' 필터로 이동
  - 알림 표시: "사용자가 승인되었습니다. 로그인이 가능합니다."

- **성공 시 (거부)**
  - 거부 완료 메시지 표시
  - 거부 사유 확인 표시
  - 사용자 목록 자동 갱신
  - 거부된 사용자가 '비활성' 상태로 변경
  - 알림 표시: "사용자가 거부되었습니다."

- **실패 시 (권한 없음)**
  - 권한 부족 안내 메시지 표시
  - 대시보드 페이지로 리디렉션

- **실패 시 (거부 사유 미입력)**
  - 거부 사유 필수 입력 안내 메시지
  - 거부 사유 입력 필드 강조
  - 처리 중단

- **실패 시 (이미 처리된 사용자)**
  - 이미 승인/거부된 사용자임을 안내
  - 현재 상태 표시
  - 사용자 목록 갱신

- **실패 시 (서버 오류)**
  - 시스템 오류 안내 메시지 표시
  - 재시도 버튼 제공

### 9.4 엣지케이스
- 동시에 여러 관리자가 같은 사용자 처리: 먼저 처리한 관리자의 작업 우선, 나중 처리 시 "이미 처리됨" 안내
- 관리자가 자신의 계정 상태 변경 시도: 자기 자신은 수정 불가 안내
- 승인/거부 처리 중 네트워크 끊김: 트랜잭션 롤백, 재시도 안내
- 대량의 pending 사용자: 페이지네이션 및 검색 기능 제공
- 거부 사유가 매우 긴 경우: 최대 길이 제한 (예: 500자)
- 사용자 정보가 불완전한 경우: 경고 표시하되 승인/거부 가능
- 만료된 토큰: 자동 로그아웃 후 로그인 페이지로 리디렉션
- 일괄 승인/거부 기능 사용 시: 확인 팝업 표시 후 처리

## 10. 사용자 프로필 관리

### 10.1 입력
- 로그인된 사용자가 프로필 페이지(/profile) 접근
- 개인정보 수정 탭 또는 비밀번호 변경 탭 선택
- **개인정보 수정 시**
  - 이름, 소속 부서, 직책 정보 수정
  - 저장 버튼 클릭
- **비밀번호 변경 시**
  - 현재 비밀번호 입력
  - 새 비밀번호 입력
  - 새 비밀번호 확인 입력
  - 변경 버튼 클릭

### 10.2 처리
- **권한 확인**
  - Django 세션 인증 검증
  - 로그인 상태 확인
  - (사유: MVP 범위에서는 Django 기본 세션 인증 사용)

- **현재 사용자 정보 조회**
  - 토큰에서 사용자 ID 추출
  - 데이터베이스에서 사용자 정보 조회
  - 사용자 정보 반환 (이메일, 이름, 소속 부서, 직책, role, 가입일)

- **개인정보 수정 처리**
  - 입력값 검증
    - 필수 항목(이름, 소속 부서, 직책) 누락 여부 확인
    - 이름, 소속 부서, 직책 길이 제한 확인
  - 변경 사항 확인
    - 기존 정보와 비교
    - 변경된 항목만 업데이트
  - 데이터베이스 업데이트
    - 사용자 정보 업데이트
    - 수정 일시 기록

- **비밀번호 변경 처리**
  - 입력값 검증
    - 필수 항목 누락 여부 확인
    - 새 비밀번호 정책 검증 (최소 4자)
    - 새 비밀번호 일치 여부 확인
  - 현재 비밀번호 검증
    - 데이터베이스의 비밀번호 해시와 비교
    - 불일치 시 프로세스 중단
  - 새 비밀번호와 현재 비밀번호 동일 여부 확인
    - 동일하면 의미 없는 변경으로 안내
  - 비밀번호 업데이트
    - 새 비밀번호 해싱 처리
    - 데이터베이스 업데이트
    - 비밀번호 변경 일시 기록

### 10.3 출력
- **페이지 로드 시**
  - 현재 사용자 정보 표시
  - 이메일은 수정 불가 (읽기 전용)
  - role, 가입일 정보 표시 (읽기 전용)

- **성공 시 (개인정보 수정)**
  - 수정 완료 메시지 표시
  - 변경된 정보로 페이지 갱신
  - 상단 네비게이션바의 사용자 정보도 갱신

- **성공 시 (비밀번호 변경)**
  - 비밀번호 변경 완료 메시지 표시
  - 모든 비밀번호 입력 필드 초기화
  - 선택적: 자동 로그아웃 후 재로그인 안내

- **실패 시 (권한 없음)**
  - 로그인 페이지로 리디렉션
  - 세션 만료 안내

- **실패 시 (입력값 오류 - 개인정보)**
  - 해당 필드에 오류 메시지 표시
  - 입력한 데이터 유지

- **실패 시 (현재 비밀번호 불일치)**
  - 현재 비밀번호 오류 메시지 표시
  - 현재 비밀번호 필드 강조
  - 다른 필드 초기화

- **실패 시 (새 비밀번호 정책 위반)**
  - 비밀번호 정책 안내 메시지 표시 (최소 4자)
  - 새 비밀번호 필드 강조

- **실패 시 (새 비밀번호 불일치)**
  - 비밀번호 확인 불일치 메시지 표시
  - 새 비밀번호 확인 필드 강조

- **실패 시 (새 비밀번호가 현재와 동일)**
  - 새 비밀번호가 현재 비밀번호와 동일함을 안내
  - 새 비밀번호 필드 강조

- **실패 시 (서버 오류)**
  - 시스템 오류 안내 메시지 표시
  - 재시도 안내

### 10.4 엣지케이스
- 변경 사항 없이 저장: "변경 사항이 없습니다" 안내
- 특수문자가 포함된 이름/부서명: 허용 가능한 특수문자 범위 정의 및 검증
- 매우 긴 입력값: 각 필드별 최대 길이 제한 (이름 50자, 부서 100자, 직책 50자)
- 동시에 여러 탭에서 정보 수정: 최신 정보로 업데이트, 충돌 방지
- 비밀번호 변경 중 네트워크 끊김: 트랜잭션 롤백, 재시도 안내
- 만료된 토큰: 자동 로그아웃 후 로그인 페이지로 리디렉션
- 이메일 변경 시도 (UI에서 막혀있지만 API 직접 호출): 이메일 변경 불가 응답
- role 변경 시도 (UI에서 막혀있지만 API 직접 호출): role 변경 불가 응답, 관리자만 가능 안내

## 11. 로그아웃

### 11.1 입력
- 로그인된 사용자가 네비게이션 바의 사용자 메뉴 클릭
- 로그아웃 버튼 클릭
- 확인 팝업에서 확인 버튼 클릭 (선택적)

### 11.2 처리
- **권한 확인**
  - Django 세션 인증 검증
  - 로그인 상태 확인
  - (사유: MVP 범위에서는 Django 기본 세션 인증 사용)

- **세션 종료 처리**
  - 클라이언트 측 토큰 삭제
    - 로컬 스토리지 또는 쿠키에서 JWT 토큰 제거
    - 세션 스토리지 정리
  - 서버 측 처리 (선택적)
    - 토큰 블랙리스트 등록 (보안 강화 시)
    - 로그아웃 시간 기록

- **상태 초기화**
  - 애플리케이션 전역 상태 초기화
  - 캐시된 사용자 데이터 삭제

### 11.3 출력
- **성공 시**
  - 로그아웃 완료 메시지 표시 (선택적)
  - 로그인 페이지(/login)로 리디렉션
  - 모든 인증 정보 삭제 확인

- **실패 시 (이미 로그아웃 상태)**
  - 로그인 페이지로 리디렉션

- **실패 시 (서버 오류)**
  - 클라이언트 측 토큰은 삭제
  - 로그인 페이지로 리디렉션
  - 에러 로그 기록

### 11.4 엣지케이스
- 이미 만료된 토큰으로 로그아웃 시도: 정상 처리하고 로그인 페이지로 이동
- 여러 탭에서 동시 로그인 상태: 한 탭에서 로그아웃 시 모든 탭에서 로그아웃 처리
- 로그아웃 중 네트워크 끊김: 클라이언트 측 토큰 삭제 후 로그인 페이지 이동
- 뒤로가기로 이전 페이지 접근 시도: 토큰 없으면 로그인 페이지로 리디렉션
- 작업 중인 데이터가 있을 때 로그아웃: 확인 팝업으로 저장되지 않은 작업 경고 (선택적)

## 12. 데이터 업로드 이력 조회

### 12.1 입력
- 로그인된 사용자가 데이터 히스토리 페이지(/data/history) 접근
- 필터 옵션 선택
  - 업로드 기간 선택 (시작일, 종료일)
  - 데이터 타입 선택 (전체, 학과별 KPI, 논문 게재 목록, 연구비 집행 데이터, 학생 명단)
  - 업로드 상태 선택 (전체, 성공, 실패)
  - 업로드자 검색 (관리자만)
- 특정 이력 항목 클릭
- 상세 정보 보기

### 12.2 처리
- **권한 확인**
  - Django 세션 인증 검증
  - 사용자 role 확인
  - 일반 사용자는 자신의 업로드 이력만 조회 가능
  - 관리자는 전체 이력 조회 가능

- **필터 파라미터 처리**
  - 기본값: 최근 1개월, 전체 데이터 타입, 전체 상태
  - 사용자가 선택한 필터값 적용

- **업로드 이력 조회**
  - 데이터베이스에서 업로드 이력 조회
  - 필터 조건 적용
  - 사용자 권한에 따라 조회 범위 제한
    - 일반 사용자: user_id가 본인인 이력만
    - 관리자: 전체 이력
  - 정렬: 최신순

- **이력 정보 집계**
  - 총 업로드 건수
  - 성공/실패 건수 및 비율
  - 데이터 타입별 업로드 건수
  - 총 업로드된 데이터 행 수

- **상세 정보 조회 (특정 이력 클릭 시)**
  - 업로드 이력 ID로 상세 정보 조회
  - 파일명, 파일 크기, 업로드 일시
  - 업로드자 정보
  - 데이터 타입
  - 처리 결과 (성공/실패)
  - 저장된 데이터 행 수 (성공 시)
  - 오류 메시지 (실패 시)
  - 오류 로그 다운로드 링크 (실패 시)

### 12.3 출력
- **성공 시 (목록 조회)**
  - 필터 옵션 패널 표시
  - 주요 통계 카드 표시 (총 업로드 건수, 성공률, 최근 업로드 일시)
  - 업로드 이력 목록 테이블 표시
    - 업로드 일시, 파일명, 데이터 타입, 업로드자, 상태, 저장된 행 수
    - 상태별 색상 표시 (성공: 녹색, 실패: 빨간색)
    - 정렬 기능
    - 페이지네이션
  - 데이터 타입별 통계 차트 (파이 차트)
  - 시간별 업로드 추이 차트 (라인 차트)

- **성공 시 (상세 정보 조회)**
  - 상세 정보 모달 또는 패널 표시
  - 모든 업로드 정보 표시
  - 성공 시: 데이터 미리보기 제공
  - 실패 시: 오류 메시지 및 오류 로그 다운로드 버튼
  - 동일 파일 재업로드 버튼
  - 해당 데이터 시각화 페이지로 이동 버튼 (성공한 경우)

- **실패 시 (권한 없음)**
  - 로그인 페이지로 리디렉션
  - 세션 만료 안내

- **실패 시 (데이터 없음)**
  - 업로드 이력 없음 안내 메시지
  - 데이터 업로드 페이지로 이동 링크 제공 (권한 있는 경우)

- **실패 시 (서버 오류)**
  - 시스템 오류 안내 메시지 표시
  - 재시도 버튼 제공

### 12.4 엣지케이스
- 필터 조합으로 결과 없음: 필터 조건 완화 제안 또는 초기화 옵션
- 매우 오래된 이력 조회: 기간 제한 안내 (예: 최근 1년)
- 대용량 이력 데이터: 페이지네이션 및 가상 스크롤링
- 삭제된 사용자의 업로드 이력: 업로드자를 "삭제된 사용자" 또는 이메일로 표시
- 오류 로그 파일이 없는 경우: "로그 파일 없음" 안내
- 파일명이 매우 긴 경우: 축약 표시 및 툴팁으로 전체 이름
- 업로드 중인 파일 (진행 중 상태): "처리 중" 상태 표시 및 진행률 (가능한 경우)
- 만료된 토큰: 자동 로그아웃 후 로그인 페이지로 리디렉션
- 동시에 여러 파일 업로드 시: 각 파일별 개별 이력 생성
