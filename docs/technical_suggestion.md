### **대학교 데이터 시각화 대시보드 MVP 기술 스택 (최종안)**

본 문서는 '대학교 데이터 시각화 대시보드' 프로젝트의 **MVP(최소 기능 제품) 개발** 단계를 성공적으로 완수하기 위한 최종 기술 스택을 정의합니다. 초기 제안의 탁월한 비전을 유지하되, **개발 속도 극대화**와 **신속한 시장 검증**이라는 MVP의 핵심 목표에 맞춰 아키텍처를 전략적으로 최적화했습니다.

#### **추천 기술 스택 요약**

| 구분 | 기술 | 선정 이유 |
| :--- | :--- | :--- |
| **뷰(View) 계층** | **Django 템플릿 + Chart.js** | 프런트엔드와 백엔드를 통합하여 별도의 API 개발, 상태 관리, 빌드 과정의 복잡성을 제거하고 **MVP 개발 속도를 압도적으로 향상**시킵니다. |
| **백엔드** | **Python, Django, Pandas** | 데이터 처리(Pandas)와 웹 개발(Django)의 강력한 조합을 유지하며, 단일 프레임워크 내에서 모든 로직을 처리하여 **개발 효율을 극대화**합니다. |
| **데이터 업로드** | **Django Admin 기본 기능** | 별도의 UI 개발 없이, Django가 기본 제공하는 강력한 관리자 페이지를 즉시 활용하여 **핵심 기능(파일 업로드)을 0에 가까운 비용으로 구현**합니다. |
| **데이터베이스** | **PostgreSQL** | 수십 년간 검증된 안정성과 데이터 무결성을 보장하며, 클라우드 환경(Railway)과의 완벽한 호환성을 제공합니다. |
| **배포/인프라** | **Railway** | 소스 코드 푸시만으로 빌드부터 배포까지 전 과정을 자동화하여, 인프라 관리 부담 없이 개발에만 집중할 수 있는 최신 PaaS 환경입니다. |

#### **기술 스택 상세 설명**

**가. 뷰(View) 계층: Django 템플릿 + Chart.js**

사용자가 마주하는 대시보드 화면을 구현합니다. MVP 단계에서는 React 대신 Django의 내장 템플릿 시스템을 활용하여 백엔드와 뷰를 통합합니다.

*   **신속한 개발:** 백엔드 로직과 화면 렌더링이 하나의 프로젝트, 하나의 언어(Python)와 템플릿 언어로 이루어집니다. 이는 API 명세 정의, CORS 정책 설정 등 분리된 환경에서 발생하는 부가적인 작업을 제거하여 개발자가 핵심 로직에만 집중하게 해줍니다.
*   **단순성과 직관성:** Chart.js는 가볍고 사용하기 쉬운 자바스크립트 차트 라이브러리입니다. Django 뷰가 전달한 데이터를 템플릿 내에서 몇 줄의 코드로 손쉽게 동적 차트로 변환할 수 있어 빠른 시각화 구현에 최적화되어 있습니다.
*   **낮은 학습 곡선:** 새로운 프런트엔드 프레임워크 학습 없이, Django 개발자라면 누구나 즉시 화면 개발에 참여할 수 있습니다.

**나. 백엔드: Python, Django + Pandas**

데이터 처리, 사용자 인증, 데이터베이스 연동 등 서버 측의 모든 로직을 담당하는 핵심부입니다.

*   **강력한 데이터 처리:** 프로젝트의 핵심인 엑셀 파일 파싱 및 데이터 정제는 데이터 과학의 표준인 Python과 Pandas 라이브러리를 통해 변함없이 강력하고 효율적으로 처리됩니다.
*   **'초고속' 기능 구현:** Django의 '배터리 포함(Batteries-included)' 철학을 극대화합니다. 특히 **Django Admin**은 모델을 정의하는 것만으로 데이터 생성, 조회, 수정, 삭제(CRUD)가 가능한 완벽한 관리자 페이지를 자동으로 생성해 줍니다. 이를 통해 MVP의 핵심 기능인 '엑셀 파일 업로드 인터페이스'를 **단 몇 분 만에 구현**할 수 있습니다.
*   **통합된 로직:** Django 뷰 함수가 데이터베이스 조회, 비즈니스 로직 처리, 그리고 최종 HTML 렌더링까지의 모든 과정을 책임집니다. 이는 코드 흐름을 단순하고 명확하게 만들어 빠른 개발과 디버깅을 가능하게 합니다.

**다. 데이터베이스 및 배포 환경: PostgreSQL on Railway**

데이터를 영구 저장하고 애플리케이션을 외부에 서비스하는 기반 환경입니다. 이 조합은 초기 제안의 탁월한 선택을 그대로 유지합니다.

*   **신뢰성의 PostgreSQL:** 데이터의 무결성과 안정성이 중요한 이번 프로젝트에 가장 적합한 오픈소스 관계형 데이터베이스입니다.
*   **편리함의 Railway:** Github 저장소에 코드를 푸시하는 것만으로 Django 애플리케이션과 PostgreSQL 데이터베이스의 배포 및 연동이 자동으로 처리됩니다. 개발자는 인프라 걱정 없이 오직 코드에만 집중할 수 있습니다.

#### **MVP 개발 및 배포 전체 흐름**

1.  **모델 정의:** Django `models.py`에 업로드할 파일과 정제된 데이터를 저장할 테이블 구조를 정의합니다.
2.  **관리자 페이지 활성화:** `admin.py`에 단 몇 줄의 코드로 정의된 모델을 등록합니다.
3.  **데이터 업로드:** 관리자는 배포된 사이트의 `/admin` 페이지에 접속하여 별도 개발된 UI 없이 엑셀 파일을 업로드합니다.
4.  **데이터 처리 및 저장:** 파일이 업로드되면, Django 시그널 또는 커스텀 로직이 이를 감지하여 Pandas로 데이터를 파싱 및 정제한 후, PostgreSQL 데이터베이스에 저장합니다.
5.  **시각화 데이터 전달:** 사용자가 대시보드 페이지에 접속하면, Django 뷰가 DB에서 데이터를 조회하여 Chart.js가 이해할 수 있는 형태로 가공한 뒤 템플릿에 전달합니다.
6.  **시각화:** 브라우저는 Django 템플릿으로부터 HTML과 데이터를 받아, Chart.js를 통해 동적인 그래프를 화면에 렌더링합니다.
7.  **자동 배포:** 이 모든 과정이 담긴 소스 코드를 Github에 푸시하면, Railway가 이를 자동으로 감지하여 빌드 및 배포를 수행하고, 즉시 서비스가 업데이트됩니다.
